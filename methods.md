# methods.md · 分析方法论与工具

所有方法论使用时，铁律-1作为元方法贯穿——每次分析都问"人类顶尖总和能做到什么，我达到了吗，超越了吗，还能更好吗"。

## §1 信息检索（模块A）

假设驱动（先形成可证伪假设再搜索，不广撒网）→定向搜索→路径受阻时系统性切换（换信息源/换时间维度/换抽象层级，至少三条不同路径才能说"确认无法获取"）→一手来源优先。搜到后的评估见模块B（`expression.md §2`事实校验）和模块C（`expression.md §3`来源可信度分层）。

## §2 分析方法论

元规则：假设先行搜索验证——搜索前先形成可证伪假设，假设被推翻是最有价值的发现。贯穿以下所有方法。

**常见搭配（不是限制，临场组合优先）：** 判断归因→因果链逐环压力测试+因果归因横向对比；解释竞争→竞争假说法；评估策略→因果链追问法+约束二分法；涉及两方关系→关系判断双侧推演；多方互动→博弈论框架；预判长期效果→系统动力学；评估证据权重→贝叶斯更新；检验核心判断→红队思维。

**竞争假说法。** 强制生成2-3个互相竞争的解释，逐条追踪证据权重，攻防后收敛。两个假说始终无法区分→差异本身是洞察。

**因果链追问法。** (1)好处→(2)代价→(3)代价随时间增长吗？不可逆？→(4)对象自己意识到了吗？追到策略的内在极限或对象盲区才算终点。

**约束二分法。** 技术约束（不可能改）vs意愿约束（可以改但选择不改）？有先例突破→意愿约束。意愿约束揭示利益排序。

**因果链逐环压力测试。** 每个A→B跳跃：充分条件测试（有A没走向B的反例？）、必要条件测试（没A走向了B？）、时间线测试（A在B之前？）。机械执行不依赖直觉。

**因果归因横向对比。** 任何"X导致Y"写入核心判断前，至少做一组控制变量横向对比。越有叙事张力的归因越需要对比。

**关系判断双侧推演。** 涉及两方的判断，必须双侧各推一遍。单侧推演最多对一半。

**博弈论框架。** (1)画收益矩阵/博弈树 (2)找纳什均衡 (3)判断博弈类型（囚徒困境/协调/鸡博弈） (4)找改变博弈结构的杠杆（改收益/改信息/改规则/改参与者）。

**系统动力学。** (1)画因果回路图（正反馈/负反馈） (2)标注延迟 (3)识别存量和流量。适用于预判政策效果、理解"为什么明显该改但没改"。

**贝叶斯更新。** (1)写下先验和置信度 (2)每条新证据问：X为真看到这个的概率/X为假看到的概率，比值=诊断力 (3)诊断力高的大幅更新，低的几乎不更新 (4)主动搜索能推翻当前判断的证据。

**红队思维。** (1)写下核心判断 (2)强制构造最有力的替代解释 (3)证据攻防 (4)结果：当前胜出/替代胜出/无法区分（标注核心不确定性）。详细操作指南见`quality.md §5`红队自检。

## §3 经验积累（模块F）

**反模式库：** 每条含表现→根因→检测→修正→实战来源。

**正面模式库：** 每条含表现→为什么有效→适用条件→实战来源。只记纠正不记成功会导致经验库偏向回避而非趋近——不犯旧错但也不重复好做法。用户确认的非显而易见的好做法值得记录。

**运行时过拟合监控：**
- F-R1 输入类型覆盖：连续5次输入类型高度相似→标注"样本偏窄"
- F-R2 规则失效信号：新类型输入时多条规则不适用/产出质量明显低/修正频率高→先判断是补规则还是改规则
- F-R3 过度打磨：连续两轮措辞优化而非结构性改进→打磨终止
- F-R4 规则膨胀：每次加规则记录来自哪个bad case。"只有这一个输入"→过拟合补丁。规则总量每增20%精简review。

## §4 可复制范式

**专业知识库 + 结构化分析 + AI生成 + 标准化输出 = 专业级产品。** 每个新场景只需替换知识库和分析维度。

**已有代码 + AI Skill + Agent平台 + 通道交互 = 自动化执行系统。**

**像素级算法流水线 + 交叉验证 + 多层防御 + 标准化输出 = 视频/图像→结构化文档。**

**领域知识 + handbook方法论 + 多文件Skill + 质量锻造 = 专业领域Skill工厂。**

**专业内容 + 结构化视觉标注 + 独立渲染管线 = 专业级电子出版物。** 内容层和渲染层遵循P1三层分离。
