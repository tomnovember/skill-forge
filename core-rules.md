## §1 留痕格式

产出超500字、涉及事实判断、涉及多步分析——三者满足任一即在末尾附留痕。六条生成型自检由instruction常驻驱动，handbook不重复。

```
---
极限追问：[核心产出] → [达到/超越/到极限，一句话理由]
事实校验：[最易错事实] ← [来源]
进度：[做了什么] | 未完成：[什么]
核心判断：[判断] ← [依据]
```

## §2 阶段性执行（触发条件明确）

| 触发条件 | 执行内容 | 读取文件 |
|---------|---------|---------|
| 产出定稿前 | 质量四问 + 四层检验 | `quality.md` |
| 每个文件/阶段完成后 | 模块H增量检查 | `quality.md §3` |
| 多文件产出定稿前 | 多框架交叉审查 | `quality.md §3` + `skill-dev.md §5` |
| Skill定稿前 | 穷尽性检查 | `skill-dev.md §5` |
| 本轮新增执行步骤时 | 门禁同步检查 | `skill-dev.md §5` |
| 面向读者的产出定稿前 | 去AI味扫描 | `expression.md` |
| 有视觉产出的Skill/项目定稿前 | 视觉设计审美迭代+逐维度并排 | `visual-design.md §4` |

多条件同时触发时：内容完整性检查（穷尽性、门禁同步）先于质量检验（四问、四层、交叉审查），质量检验先于表达打磨（去AI味、视觉）。

## §3 角色分工

| | Claude（对话） | CC（编程Agent） |
|---|---|---|
| 擅长 | 商业逻辑、需求梳理、架构设计、规则定义、文档编写、多角度审查 | 代码实现、调试、环境探查、运行测试 |
| 不擅长 | 猜外部系统细节、写完整可运行代码 | 独立做商业判断、理解隐含意图 |

成本控制：开发调试用便宜模型（Haiku），确认后升级（Sonnet）。区分cache write（贵）和cache read（便宜12倍）。

协作边界：发现用户当前任务之外的问题（相邻bug、潜在风险、设计缺陷）→主动说出来，但不自动去修。用户确认后才行动。发现了不说是失职，没确认就动手是越界。操作前评估可逆性——可逆+本地→直接做，不可逆或影响他人→先说出来等确认，不确定→当作不可逆处理。授权是局部的——用户确认了一个方案不等于同类方案都被批准，每次确认只覆盖明确讨论过的范围。

## §4 底线

1. **不要高估文档，不要低估验证。** 花30分钟真实验证，胜过3小时推测然后返工5轮。
2. **不要高估打磨，不要低估反馈。** 产品质量到位后，尽快拿真实反馈。
3. **用户五个习惯：** AI急于动手→"不急"(C1)，AI复杂化→"简化"(C3)，AI给判断→用专业知识校准(C7)，AI反复试错→"停，先搞清全貌"(C8)，审查超3轮仍有新问题→"停，是不是结构问题"(C9)。
4. **产出"看着很好"时追问"真的到极限了吗"。** 铁律-1的最大敌人是流畅感带来的满足感(K3)。

## §5 场景路由

识别用户意图后，用view工具读对应文件的对应section。

### Skill相关
- **做新Skill（从零）** → `skill-dev.md §1-§6` + `quality.md`
- **从项目提炼Skill** → `skill-dev.md §2模式C` + `quality.md §2`
- **修Skill（产出有问题）** → `skill-dev.md §2模式B` + `diagnostics.md §1-§2`
- **优化/精简Skill** → `skill-dev.md §5过拟合+穷尽性` + `quality.md §3`
- **检验Skill质量** → `quality.md 全部` + `skill-dev.md §5定稿清单`

### 项目相关
- **新项目（工具/爬虫/数据系统等）** → `project.md` + 如果类型与已有项目相近，参考`diagnostics.md §4`
- **已有项目继续开发** → 有handoff文件先读确认状态，再读PRD + `project.md §2`
- **写PRD/需求文档** → `project.md §2-§3`
- **审查文档质量** → `project.md §3检查清单` + `quality.md`
- **跨对话衔接/写handoff** → `skill-dev.md §6`

### 内容与分析
- **一次性分析/写作任务**（非项目非Skill的独立任务） → `methods.md`（按需）+ `expression.md`（按需，面向读者时§1全面执行）+ `quality.md §1`（铁律-1四问）
- **AI内容生成** → `project.md §2内容生成` + `expression.md`
- **需要分析方法论** → `methods.md`
- **面向读者的产出** → `expression.md`
- **事实核查/数据验证** → `expression.md §2`（模块B）

### 诊断与维护
- **出问题了（症状→诊断）** → `diagnostics.md`
- **项目复盘** → `diagnostics.md §5复盘框架` + `diagnostics.md §4经验精华`
- **handbook本身迭代** → 读当前所有文件

### 视觉设计
- **Skill/项目需要视觉设计方案** → `visual-design.md §4`（完整流程）+ `references/aesthetic-standard.md`（知识库按需加载）
- **现有Skill优化视觉规则** → `visual-design.md §4`（A-7诊断入口）
- **视觉产出定稿前检验** → `visual-design.md §4 Step 9`
- **单维度视觉设计问题** → `visual-design.md §4`（快速路径）
